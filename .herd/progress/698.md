# Issue #698 — Fix CI failure on batch 90 (codex-provider-chatgpt-subscription-auth)

## Root cause
CI's "Check workflow drift" step runs `herd init --check`, which renders the
embedded `docker-compose.herd.yml.tmpl` and compares it to the committed
`docker-compose.herd.yml`. Batch 90 updated the template to add Codex
subscription auth (CODEX_AUTH_JSON / CODEX_ACCESS_TOKEN / HERD_CODEX_KEEPALIVE_INTERVAL
env vars, a `codex-auth` volume, and the replicas comment block), but the repo's
own committed `docker-compose.herd.yml` was never regenerated → drift → CI fail.

`herd init` does not fix the working tree on its own because `commitInitFiles`
commits the regenerated file on a throwaway `herd/init-<version>` branch and then
`git checkout -`s back, reverting the working copy.

## Fix
Regenerated `docker-compose.herd.yml` from the current template (canonical render
for owner/repo `Herd-OS/herd`, provider `claude`, codex_replicas default 1).
